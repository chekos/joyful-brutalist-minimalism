# Design decisions

Use a decision record when an accepted choice changes the design language's
meaning, ownership boundaries, portability, or cross-medium workflow. Keep
temporary options and active work in GitHub issues.

Copy [`template.md`](template.md), assign the next four-digit number, and link
the authorizing issue and resulting pull request. Update the authority that owns
the decision; the record explains the choice but does not replace that source.
